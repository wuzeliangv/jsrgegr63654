# 大姨子面板 (Dayizi Panel) 管理后台使用指南

本指南旨在全面介绍大姨子面板管理后台（`/admin`）的各项核心功能、配置流程、自动化策略与运维排查规范。

---

## 🚀 首次使用流程

1. **一键部署与域名绑定**：使用一键安装脚本部署面板，配置面板域名并申请 Nginx HTTPS 证书。
2. **注册超级管理员**：访问 `https://panel.yourdomain.com/auth/email-register` 注册**首个账号**，系统会自动将其赋予系统超级管理员身份。
3. **配置 SMTP 邮件服务**：登录管理员账号进入 `/admin` ➔ **设置** ➔ 填写 SMTP 邮件服务器配置并发送测试邮件。
4. **开放用户注册**：配置好 SMTP 后，其他新用户注册时即可正常接收邮件验证码。

---

## 🛠️ 后台核心模块

### 1. 节点编排与部署
- **协议智能部署**：支持一键向远程 Linux VPS 部署 **VLESS Reality** (TCP/WS)、**Shadowsocks** 以及 **Hysteria 2**，可选择密码或 SSH 私钥认证。
- **SOCKS5 落地挂载**：支持为节点配置外部或本地 SOCKS5 落地出口（例如结合分布式 NodePool 节点池使用），实现流量出口 IP 落地。
- **AWS 云集成与一键换 IP**：
  - 支持绑定 AWS EC2 与 Lightsail 账号。
  - 支持针对 IP 被墙的节点一键自动重换公网 IP（含 Lightsail 静态 IP 与 Wavelength 弹性 IP）。
- **运维控制**：支持远程同步配置、一键重启远程 Xray / Hysteria 2 核心以及服务联通性检测。

### 2. 用户与配额管理
- **灵活分组**：可划分家宽组、SVIP、SSVIP 等组别，针对不同组别设置每月流量上限、到期时间、节点可见性与重置周期。
- **状态管控**：支持一键封禁、解冻、重置订阅 Token/UUID 及删除用户。
- **树状邀请关系**：清晰呈现用户邀请路径与裂变层级。

### 3. 流量与风控审计
- **全方位统计**：提供按用户、按节点维度的 1 小时/24 小时/7 天/30 天流量明细及排行。
- **订阅风控引擎**：支持记录订阅访问 IP、设备 UA 及频次，智能识别滥用行为与 Token 异常共享。
- **安全审计日志**：全量记录管理员操作、敏感设置修改与节点变更履历，过期日志自动清理。

### 4. 自动化运维策略
- **节点被墙自动换 IP**：定时任务后台检测节点连通性，被墙时触发警告并可选自动替换 AWS 绑定的 IP。
- **TG 不活跃自动冻结**：开启后，超过 30 天未在 Telegram 签到的用户将被自动冻结；用户重新签到后自动解冻。
- **密钥与订阅 Token 轮换**：可设置周期自动为指定用户组重置 UUID 或订阅 Token。

### 5. 数据库备份与恢复
- **一键备份**：生成带时间戳的 SQLite 数据库快照并支持下载。
- **数据库恢复**：支持从历史快照一键恢复数据库。
- **自动备份**：系统每日凌晨自动创建备份并保留最近 7 天的记录。

## 🤖 Telegram Bot 功能清单

需在 `.env` 中配置 `TG_BOT_TOKEN`。

| 指令 | 功能说明 |
|---|---|
| `/start` | 启动机器人并呈现主功能菜单（支持深层链接绑定账号） |
| `/bind <token>` | 绑定面板账户与 TG 账号 |
| `/checkin` | 每日签到领随机流量（连续签到自动提升用户组等级） |
| `/me` | 查询个人配额、已用流量、到期时间与等级状态 |
| `/sub` | 快速获取专属节点订阅链接 |
| `/traffic` | 查询当前计费周期的详细流量占比 |
| `/nodes` | 查看系统当前可用节点状态与延迟概览 |
| `/lucky` | 启动 WebApp 幸运大转盘抽奖 |
| `/flip` | 启动 WebApp 每日翻卡小游戏 |
| `/rps` | 启动 WebApp 猜拳对决小游戏 |
| `/adminstats` | 管理员总览面板（仅管理员可见） |

## 📊 节点探针 (Monitor)

所有已登录用户均可访问 `/monitor` 页面（不含敏感密钥信息）：
- **机器资源监控**：实时呈现节点服务器 CPU 占用、内存消耗、磁盘空间与系统负载。
- **网络流量分析**：实时显示上行/下行网速与历史流量曲线。
- **服务存活状态**：实时标注 Agent 通信状态及 Xray/Hysteria 内核运行状态。

## 🔍 故障排查与运维命令

```bash
# 1. 检查面板 PM2 运行状态
pm2 status

# 2. 查看面板最新运行日志
pm2 logs dayizi-panel --lines 100

# 3. 检查与重载 Nginx 反向代理
nginx -t && systemctl reload nginx

# 4. 在远程节点上检查 Node Agent 运行日志
journalctl -u vless-agent -f -n 50

# 5. 在远程节点上检查 Hysteria / Xray 服务状态
systemctl status hysteria-server
systemctl status xray
```
